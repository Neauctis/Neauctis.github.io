Image TV

  1. Accueil
  2. Image TV

Problématique

Des téléviseurs connectés se trouvent sur 3 sites différents du Groupe et ont pour objectif d’afficher des informations concernant l’entreprise. Le problème est qu’il était difficile de mettre à jour les informations qui apparaissent sur ces téléviseurs. Face à cette problématique, j’ai donc proposé de créer une application que la chargée de communication pourra utiliser pour ajouter, modifier ou supprimer des images qui seront stockées dans une base de données. Il faut également créer des sites Web capables d’afficher les images stockées dans la base de données sous forme de diaporama. Chaque site Zekat en possédera un différent. La base de données ainsi que les sites Web seront stockés sur un serveur Linux (Debian 12) que je vais devoir créer.

J’ai donc commencé le lundi 3 juin le projet en commençant par le développement de l’application.

Application

J’ai décidé de développer en C# car c’est ce que nous utilisons en cours et je me projetais déjà sur quels outils et quelles méthodes vues en cours utiliser. Je ne voyais donc pas d’inconvénient à utiliser du C#.

J’ai donc réfléchi à ce que j’allais mettre sur l’application, à la façon dont l’utilisateur pourra ajouter des images, aux informations qu’il devra renseigner, à ce qu’il aura besoin de voir sur l’application, à la sécurité à mettre en place, etc.

Fenêtre "Connexion" :

J’ai donc commencé par créer une fenêtre de connexion qui demandera à l’utilisateur un mot de passe (il n’y a qu’un seul mot de passe stocké et hashé dans la base de données).

Fenêtre "Menu Principale" :

Lorsque l’utilisateur entre le bon mot de passe, il accède à la page du menu principal.

Il y a une barre de navigation en haut qui donne 3 options à l’utilisateur.

  • Ajouter

    • Permet d’ajouter dans la base de données des images se trouvant sur le PC de l’utilisateur.

  • Modifier

    • Permet de modifier le titre, la durée d’affichage en secondes, l'ordre d’affichage ainsi que le site Zekat où apparaîtra l'image.

  • Supprimer

    • Permet de supprimer les images dans la base de données.

Fenêtre "Ajouter" :

En haut de la fenêtre, il y a un tableau donnant diverses informations comme le titre de l’image, son extension, la durée d’affichage sur le site en secondes et sur quel site Zekat l'image apparaîtra. Le titre, la durée d’affichage, l’ordre d’affichage et le site d'affichage peuvent être modifiés. Lorsque l’utilisateur clique sur une information dans le tableau, il voit une prévisualisation de l’image liée à la ligne sélectionnée.

Il est possible de modifier plusieurs lignes en même temps, c'est là que la fonctionnalité "Verrouiller" entre en jeu. Cette fonctionnalité permet de verrouiller certaines valeurs afin de ne pas les modifier par erreur lors d'une modification multiple.

Fenêtre "Modifier" :

Sur cette fenêtre, il y a exactement la même disposition ainsi que les mêmes paramètres d'image que la fenêtre "Ajouter", à l'exception de la fonctionnalité permettant de changer l'ordre des images. Cette fenêtre sert à modifier les informations des images après les avoir ajoutées, en cas d'oubli ou d'erreur par exemple.

Fenêtre "Supprimer" :

Cette fenêtre a la même disposition que les deux précédentes. Elle permet de supprimer les images de la base de données.

Base de Données

Modèle Conceptuel

Table « EXTENSION » :

La table « EXTENSION » contient toutes les extensions autorisées dans la base de données. Chacune des extensions a un « idExtension », qui est un numéro lié au libellé de l’extension, c’est-à-dire son nom. Par exemple, l’extension « .png » est reliée à l'id « 3 ». Quand on veut parler de l’extension de l’image dans la base de données, on dira que son extension est « 3 ». Cela évite la redondance des données.

Table « MDP » :

La table « MDP » sert seulement à stocker le mot de passe de l’application.

Table « IMAGE » :

La table « IMAGE » contient toutes les informations à propos de l’image et stocke l’image en elle-même. Les informations contenues sont le titre, le temps d’affichage sur le site et comme mentionné précédemment, l'ID de son extension.

Table « SITE » :

La table « SITE » contient les 3 sites Zekat (Jugon-les-Lacs, Bellevigne-les-Châteaux, Beaucouzé) ainsi que leurs ID.

Table « AFFICHER » :

La table « AFFICHER » contient l'ID de toutes les images de la base de données et à quelle ID de la table « SITE » les images sont liées.

Le Site

Pour les sites, j’ai décidé de regrouper tout le code en un seul fichier : le CSS, le JavaScript, le PHP et l’HTML. J’ai pris cette décision car le code du site est très court et je ne voyais pas l’intérêt de créer des dossiers différents avec des fichiers séparés. Pour moi, cela prendrait de la place inutilement sur le serveur, donc j’ai choisi de tout regrouper. J’ai seulement mis à part les informations de connexion à la base de données pour qu’elles ne soient pas accessibles par l'utilisateur.

Les sites ont pour but d’afficher une par une les images stockées dans la base de données. Les images s’adapteront automatiquement à la résolution des téléviseurs. Lorsque toutes les images ont été affichées, le site se rafraîchit automatiquement au cas où des images sont ajoutées ou supprimées de la base de données, évitant ainsi d’avoir à rafraîchir manuellement le site sur chacune des 3 téléviseurs. Les images sont affichées pendant le nombre de secondes indiqué lors de leur ajout et ne sont visibles que sur les téléviseurs correspondant à leur ID de site.

Le Serveur

J’ai décidé de faire le serveur sur un linux Debian 12 car c’est ce qu’on utilise en cours et je ne voyais pas de contrainte m’obligeant à le faire sur un autre OS. J’ai donc configurer le serveur (Adresse IP, SSH, Sécurité etc…) puis j’ai commencé par installer MariaDB pour ma base de données. J’ai également installé PHPMyAdmin afin que cela soit plus simple à mettre en place et ainsi avoir une interface graphique. J’ai malheureusement eu un problème entre MariaDB et mon application, j’ai donc dû changer pour MySQL. Après avoir fait ma base de données, j’ai commencé à installer Apache pour y mettre mon site. Il ne me restais qu'à rendre PHPMyAdmin et le site web accessible.

Autre

Sécurité :

Les informations de connexion à la base de données (adresse IP, nom d’utilisateur, mot de passe) étaient présentes dans le code de l’application, ce qui représente un danger car si une personne accède au code de l’application, elle peut facilement s’y connecter. C’est pourquoi j’ai décidé de mettre ces informations dans un fichier « .conf » à part.

Le mot de passe pour avoir accès à l’application était également dans le code, donc toute personne ayant accès au code avait accès à l’application. J’ai donc décidé de mettre le mot de passe de l’application dans la base de données et de le hasher (crypter), ce qui rend le mot de passe illisible dans la base de données.

J’ai également séparé les informations de connexion à la base de données pour le site web.

Documentation :

Il a fallu que je crée une documentation pour que la chargée de communication comprenne le fonctionnement de l'application. Pour cela, j'ai simplement créer un document Word avec des captures d'écran et des explications étape par étape des fonctionnalités et de leur utilisation.

Bilans du Stage

Ce stage m'a appris à gérer un projet du début à la fin et avec une relative autonomie. J'ai dû réfléchir à quelles fonctionnalités inclure dans mon application pour la rendre aussi simple que possible pour les utilisateurs.

Pour améliorer mon approche du projet, j'aurais dû mieux m'organiser en établissant un planning détaillé et en consignant régulièrement mes progrès. J'ai également été confronté à plusieurs difficultés, notamment une nouvelle problématique survenue alors que je pensais être à la fin de mon projet. J'ai dû revoir et modifier l'ensemble de mon travail pour y faire face.

Ce stage m'a aussi permis d'apprendre à travailler de manière indépendante, car mon tuteur n'était présent que quelques jours par semaine.
